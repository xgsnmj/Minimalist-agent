# Multiple agents with a default agent

Minimalist Agent will initialize a Default Agent for the primary user conversation entry point while allowing Administrators to create, configure, enable, disable, and retire additional Agents. This keeps the MVP approachable for Users but preserves the product's core governance model: model selection, MCP access, sandbox capability, search capability, page read capability, process visibility, and tool authorization are configured at the Agent boundary. The MVP will not introduce an agent marketplace or public agent discovery flow.
